**CAN FD (CAN with Flexible Data-rate)** 是传统 CAN 总线（Classic CAN）的升级版。简单来说，它是一种**高速、高容量、向下兼容**的串行通讯协议设备。

如果把传统 CAN 比作一条只能跑卡车的乡村公路，那么 CAN FD 就是一条升级后的**高速公路**：在车少（仲裁段）时按限速开，在直道（数据段）时允许超速飙车，且每辆车能装的货更多。

## 1. 核心概念：为什么叫 "FD"？

"FD" 代表 **Flexible Data-rate（灵活数据速率）**。它的核心特征在于“两段式”通讯：

- **控制段（仲裁段）不变：** 为了保证多节点竞争时的稳定性，这部分的速率与普通 CAN 一样（最高 \$1Mbps\$）。

- **数据段加速：** 当总线竞争结束，开始发送实际数据内容时，频率会瞬间切换到更高的速率（通常为 \$2Mbps\$、\$5Mbps\$ 甚至 \$8Mbps\$）。

## 2. CAN FD 设备的主要特征

与普通 CAN 设备相比，CAN FD 设备有三个质的飞跃：

### **1) 更大的载荷 (Payload)**

- **普通 CAN：** 每个数据帧最多只能发 **8 字节**。

- **CAN FD：** 每个数据帧最多可以发 **64 字节**。

- *意义：* 减少了拆包和组包的麻烦，大幅提升了传输效率。

### **2) 更高的带宽 (Throughput)**

- 通过在数据段提高时钟频率，总线的实际吞吐量提升了 5-8 倍。这对于需要刷写固件（OTA）或传输大量传感器数据的场景（如摄像头、雷达）至关重要。

### **3) 更强的校验 (CRC)**

- 由于单帧数据变长了，CAN FD 引入了更复杂的 CRC 校验算法，确保在高频传输下的数据可靠性。

## 3. CAN FD 的兼容性（非常重要）

这是在 RK3568 等嵌入式开发中经常踩坑的地方：

- **向下兼容：** CAN FD 控制器可以读懂普通 CAN 报文。

- **向上不兼容：** **普通 CAN 节点无法识别 CAN FD 报文**。如果把一个普通 CAN 设备挂在正在跑 CAN FD 的总线上，普通设备会因为看不懂复杂的 FD 帧而报错，甚至导致整条总线瘫痪（Error Passive）。

## 4. 常见的 CAN FD 设备形态

在你的开发场景中，CAN FD 设备通常以以下形式出现：

1.  **SoC 集成控制器：** 比如 **RK3568** 内部就集成了支持 CAN FD 的控制器（通常在 Linux 中显示为 can0, can1）。

2.  **外置收发器（Transceiver）：** 即使 SoC 支持，也需要一个支持 FD 的收发器芯片（如 **SIT1044** 或 **TCAN1042**），才能连接到物理总线。

3.  **调试工具：** 如 USB-CAN FD 分析仪，用于在电脑上抓取和分析总线上的数据。
