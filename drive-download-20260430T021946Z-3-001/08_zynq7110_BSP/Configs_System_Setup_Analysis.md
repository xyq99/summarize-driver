# Zynq 系统全局配置与参数分析 (configs)

本文档对 `configs` 目录下的系统配置工程进行深度解析。该工程定义了 Zynq 平台的运行参数、硬件引脚功能映射以及特定业务逻辑（如水声通信）的初始状态。

---

## 一、 目录结构概述

`configs` 目录作为一个独立的配置模块，其结构如下：
- **include/**: 存放全局宏定义头文件，是系统参数的“配置库”。
- **src/**: 存放与系统配置相关的底层处理逻辑（如缓存管理、应急处理）。
- **lib/**: 编译生成的库文件存放地。
- **.project / .cproject**: Eclipse/Vitis 工程配置文件，表明这是一个可独立管理的资源工程。

---

## 二、 核心配置文件解析

### 1. configs.h (项目全局基础配置)
这是系统最核心的定义文件，涵盖了从硬件到网络的基础参数：
- **采样与算法速率**:
    - `FS_96`: 定义了 96kHz 采样率背景下的算法配置。
    - `RX_BUFF_SIZE`: 定义了接收缓冲区大小，默认为 `96*500`。
- **默认网络参数**:
    - **本机 IP**: `192.168.188.188`
    - **目标设备 (PAD) IP**: `192.168.188.100`
    - **默认端口**: `8060`
- **版本控制**: 
    - 固件标识符: `705_higspeed`
    - 版本号格式: `V1.0.0`

### 2. sot_sysConfig.h (业务逻辑参数)
该文件详细定义了水声通信及系统状态机的初始寄存器值：
- **水声物理层参数**:
    - **中心频率 (FC)**: 定义了不同模式下的中心频率（如 120kHz, 240kHz）。
    - **带宽 (BW)**: 定义了工作带宽（如 30kHz, 60kHz）。
    - **声速**: 设定为 `15000` (即 1500.0 m/s)。
- **UART 与控制模式**:
    - 默认波特率: `115200`。
    - 控制界面模式: `Ui_Mode_CMD` (命令行模式)。

### 3. autoconf.h (自动生成配置)
通常由图形化配置工具或 Kconfig 系统生成，包含了 `CONFIG_USBL_USE` (超短基线使用)、`CONFIG_SYNC_PPS` (秒脉冲同步) 等功能开关。

---

## 三、 硬件引脚映射 (GPIO/EMIO)

`configs.h` 中对 Zynq 的 EMIO 引脚进行了详细的功能绑定，这是理解软硬件交互的关键窗口：

| 引脚号 (EMIO) | 功能定义 | 描述 |
| :--- | :--- | :--- |
| 54 | `EMIO_PVDD` | PVDD 过流检测 (输入) |
| 55 | `EMIO_RXGAIN` | 接收增益控制 (输出) |
| 56 | `EMIO_MCU_BOOT` | MCU 启动控制 |
| 59 | `EMIO_SYS_POWER` | 系统断电控制 (低电平有效) |
| 60 | `EMIO_MCU_NOTIFY` | MCU I2C 通知检测 |
| 62 | `EMIO_PPS` | 秒脉冲信号输入检测 |
| 69 | `EMIO_START` | 预留启动/触发 IO |

---

## 四、 核心源码逻辑实现 (src)

该工程不仅包含宏定义，还实现了一些关键的底层支撑逻辑：

- **dgrmbuff.c (数据报文缓存管理)**:
    - 负责管理跨模块传输的数据封装与缓存区分配，确保高速数据流在处理过程中的完整性。
- **contingency.c / ctgcy.c (应急处理)**:
    - 实现了系统异常情况下的“兜底”逻辑。例如，当检测到电压异常或关键链路超时时，触发预设的安全状态切换。
- **gpioexsgl.c (GPIO 扩展信号)**:
    - 对 EMIO 引脚的读写进行了二次封装，提供了更高级别的信号处理接口，如消抖或电平触发逻辑。

---

## 五、 总结

`configs` 目录是整个固件系统的“控制中心”。它通过 `configs.h` 完成了**硬件抽象层与引脚功能的绑定**，通过 `sot_sysConfig.h` 完成了**业务逻辑参数的标准化**。在开发过程中，若需修改设备 IP、调整算法频率或重新定义 IO 功能，该目录是首要的操作对象。
