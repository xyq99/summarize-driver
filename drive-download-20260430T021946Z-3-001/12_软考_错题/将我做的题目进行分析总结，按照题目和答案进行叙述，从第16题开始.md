## 强化变式题考点与答案深度解析（续 16 题）

### 16. 计算机系统基础（错题变式加练）

- **题目概要**：在处理器流水线中，若后序指令需要用到前序指令尚未写回寄存器的执行结果，从而导致流水线必须暂停或插入气泡，这种冲突属于哪种冲突？

- **正确答案**：**数据相关（Data Hazard）**

- **核心分析**：这是对第一轮流水线冲突的精准变式。第一轮考的是“争用硬件资源（结构相关）”，而本题考的是“**数据依赖**”。当后序指令的输入取决于前序指令的输出时，就构成了数据相关（如写后读 RAW）。解决手段通常包括**旁路技术（数据前推）或插入气泡（流水线暂停）**。

### 17. 操作系统知识（错题变式加练）

- **题目概要**：为了彻底解决优先级翻转问题，高可靠性系统常采用“优先级上限协议（PCP）”，该协议的核心规则是什么？

- **正确答案**：**每个资源都有一个静态分配的优先级上限，等于可能申请该资源的所有任务中的最高优先级**

- **核心分析**：第一轮考查了基础的优先级继承协议（PIP），而本题更进一步考查了高级的**优先级上限协议（PCP）**。PCP 的核心在于**提前预防**：给互斥锁（资源）静态分配一个“上限优先级”。一旦有低优先级任务获取了该资源，其优先级会立刻被提升到该上限，这样不仅能彻底防止优先级翻转，还能**在源头上避免死锁**。

### 18. 数字电路基础（错题变式加练）

- **题目概要**：以下电路或器件中，其输出不仅取决于当前的输入信号，还与电路过去的状态相关的时序逻辑电路是？

- **正确答案**：**双向移位寄存器**

- **核心分析**：第一轮要求识别组合逻辑（全加器），本题变式要求识别**时序逻辑**。关键在于看器件内部是否包含**触发器或锁存器等记忆元件**。译码器、数值比较器和数据选择器（MUX）的逻辑输出都是瞬时随输入改变的（组合逻辑）；而**寄存器**必须在时钟边沿驱动下，结合内部原本保存的状态才能决定输出，因此是典型的时序逻辑电路。

### 19. 嵌入式系统的存储体系（错题变式加练）

- **题目概要**：在处理器访存过程中，如果因为 Cache 空间容量不足导致先前读入的数据块被置换出去，随后 CPU 再次访问该数据块时发生的未命中属于？

- **正确答案**：**容量不命中（Capacity Miss）**

- **核心分析**：第一轮考查了初次访问必定发生的“强制不命中”。本题变式考查了由于“**Cache 容量受限**”导致的失效。当程序运行所需的数据集（工作集）大于 Cache 的总容量时，数据块就会被频繁地挤出和反复调入，这种由于空间满了引起的失效就是容量不命中。

### 20. 嵌入式硬件设计（错题变式加练）

- **题目概要**：要求处理器将功耗降到最低，关闭绝大多数外设，但要求静态 RAM（SRAM）中的运行数据和内核上下文完全不丢失，应该切入哪种状态？

- **正确答案**：**停止 / 挂起模式（Stop / Suspend Mode）**

- **核心分析**：本题再次强化了低功耗电源管理模式。在嵌入式芯片（如 STM32、Rockchip 核心等）中，**SRAM 数据不丢失**是划分低功耗状态的分水岭。休眠模式省电太少；掉电/待机模式（Standby）**虽然省电到纳安级，但会切断 RAM 供电导致数据全丢；只有**停止/挂起模式（Stop）能在关闭几乎所有外设时钟的同时，维持 RAM 极低电压供电以锁存数据，是复杂嵌入式系统最常使用的挂起状态。

### 21. 知识产权与标准化（错题变式加练）

- **题目概要**：由地方政府（如省、自治区、直辖市标准化行政主管部门）制定并批准发布的标准，其代号通常表现为什么？

- **正确答案**：**DB**

- **核心分析**：第一轮考查了国家强制标准代号（GB）。本题考查了**地方标准代号（DB，即“地标”的拼音首字母）**。在我国标准体系中：**GB** 为国家标准，**HB** 等多为行业标准（如航空行业），**DB** 为地方标准，**Q/** 开头的则为企业标准。

### 22. 嵌入式软件程序设计

- **题目概要**：两个独立的硬件外设（如 DMA 控制器和 CPU 内核）频繁异步地读写同一块共享内存缓冲区，为防止编译器过度优化而使用寄存器旧值，定义指针时必须加入什么修饰符？

- **正确答案**：**volatile**

- **核心分析**：本题将第一轮中关于 volatile 的应用场景具体化到了**内核与外设（DMA）共享内存**的工程实战中。编译器为了提高效率，往往会把频繁读取的内存变量“缓存”到 CPU 的通用寄存器中。但 DMA 是在外部直接改写物理内存的，CPU 的寄存器根本感知不到，如果不加 volatile 强制每次都进行实际的内存总线读写，系统就会读到寄存器里的脏数据。

### 23. 数据结构和算法

- **题目概要**：循环队列结构中，已知数组总容量为 \$M\$，当前队头指针为 \$front\$，队尾指针为 \$rear\$。判断该队列此时为“满”的经典判定条件是什么？

- **正确答案**：**(rear + 1) % M == front**

- **核心分析**：在基于数组实现的循环队列中，如果队列空和队列满都用 rear == front 来判断，系统将无法区分。工程上最经典的解决方法是**故意留空一个数组位置**。当队尾指针往后走一步并通过模运算回绕后（即 (rear + 1) % M），如果正好迎头撞上了队头指针（front），就证明队列已经塞满了。

### 24. 网络基础知识

- **题目概要**：内网设备获取到 IP 地址后，在通过本地网关（路由器）访问互联网外部服务器时，数据包的哪个关键字段一定会发生改变？

- **正确答案**：**源 IP 地址**

- **核心分析**：局域网内部设备使用的是私有 IP（如 192.168.x.x），这类 IP 在公网上是不被路由的。当数据包跨越网关（路由器）走向互联网时，路由器必须启动 **NAT（网络地址转换）机制，将数据包头部的源 IP 地址**修改为路由器自身的公网 IP。目的 IP（外部服务器）在传输过程中不能变，否则数据就无法送达目的地。

### 25. 专业英语

- **题目概要**：看懂 RTOS 抢占式调度 manual 完形填空：“Under the preemptive priority-based scheduling algorithm, the running task will be immediately \_\_\_\_\_\_\_ if a higher-priority task becomes ready to run.”

- **正确答案**：**preempted / suspended**

- **核心分析**：这句话考查了实时操作系统的核心调度术语。在**基于优先级的抢占式（preemptive）调度算法**下，一旦有一个更高优先级的任务进入就绪态，当前正在运行的低优先级任务将会立刻被**抢占（preempted）/挂起（suspended）**，而不是被终止（terminated）或销毁（destroyed）。
