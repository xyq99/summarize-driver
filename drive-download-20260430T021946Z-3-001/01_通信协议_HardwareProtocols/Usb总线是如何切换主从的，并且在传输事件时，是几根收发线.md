USB 总线的角色切换和物理线路结构是嵌入式开发中非常基础且关键的概念。

## 1. USB 主从角色的切换机制

在传统的 USB 架构中，角色是固定的（Host 或 Device）。但为了实现同一个接口既能当主机（接 U 盘）又能当外设（接电脑），引入了 **OTG (On-The-Go)** 和最新的 **Type-C** 协议。

### 传统 Micro-USB OTG (ID 引脚检测)

在 Micro-USB 接口中，除了 VCC、GND、D+、D- 外，多了一根 **ID (Identification)** 线。

- **作为 Host (主机):** 当连接线的 ID 引脚在连接端被**接地 (GND)** 时，SoC 的 USB 控制器检测到低电平，触发中断切换为 Host 模式，并开始向总线供电（VBUS）。

- **作为 Device (从机):** 当 ID 引脚**悬空 (Float)** 时，控制器默认为 Device 模式，等待外部 VBUS 供电并进行枚举。

### USB Type-C (CC 引脚检测)

Type-C 取消了 ID 引脚，改用两根 **CC (Configuration Channel)** 线。

- **DRP (Dual Role Port):** 这种接口可以在 Host 和 Device 之间切换。

- **原理:** 接口内部有上拉电阻 (R_p) 和下拉电阻 (R_d)。

  - 想当 Host 时，连接 R_p。

  - 想当 Device 时，连接 R_d。

- 当两个 DRP 设备连接，它们会通过 CC 线进行握手（电平协商），最终一方成为 DFP (Downstream Facing Port，即 Host)，另一方成为 UFP (Upstream Facing Port，即 Device)。

## 2. 传输事件时的收发线数量

USB 的物理连线数量取决于其协议版本。USB 始终采用**差分信号 (Differential Signaling)** 来传输数据。

### USB 2.0 及以下 (2 根数据线)

无论是在控制传输、中断传输还是批量传输中，USB 2.0 仅使用 **1 对差分线**：

- **D+ 和 D-**

- **工作模式:** **半双工 (Half-Duplex)**。这意味着在同一时刻，要么是主机发给从机，要么是从机发给主机，不能同时收发。

### USB 3.0 / 3.1 / 3.2 (6 根及以上数据线)

USB 3.0（超高速 SuperSpeed）引入了真正的**全双工 (Full-Duplex)** 传输。

- **线路组成:**

  1.  **D+/D-:** 保留 1 对，用于向下兼容 USB 2.0。

  2.  **SSTX+ / SSTX-:** 1 对专门负责发送的差分线。

  3.  **SSRX+ / SSRX-:** 1 对专门负责接收的差分线。

- **工作模式:** 传输事件时，发送和接收路径在物理上是独立的，可以同时进行。

### 总结表

| 协议版本    | 数据线数量 (不含电源) | 传输模式 | 关键信号名称            |
|-------------|-----------------------|----------|-------------------------|
| **USB 2.0** | 2 根                  | 半双工   | D+, D-                  |
| **USB 3.x** | 6 根                  | 全双工   | D+/D-, SSTX+/-, SSRX+/- |
| **USB 4**   | 8 根+                 | 全双工   | 采用多通道 (Lane) 传输  |

对于你正在调试的 RK3568 或 Zynq 等平台，通常在设备树（Device Tree）中通过 dr_mode 属性（可设置为 host, peripheral, 或 otg）来手动指定或开启自动切换功能。
