**JTAG** 是 **Joint Test Action Group**（联合测试工作组）的缩写，它既是一个标准组织的名字，也是指代一种国际标准测试协议（即 **IEEE 1149.1**）。

在嵌入式开发、底层驱动和芯片设计领域，JTAG 是最核心的硬件接口之一。简单来说，它是开发者进入芯片内部的一扇“后门”。

## 1. JTAG 的核心功能

JTAG 最初是为了解决电路板上芯片引脚太多、难以进行物理测试的问题而设计的，但现在它的用途已经扩展到了更多领域：

- **硬件仿真与调试（Debug）：** 这是开发者最常用的功能。通过 JTAG，你可以让运行中的 CPU 暂停（挂起），查看寄存器数值、内存状态，并实现**单步执行**代码。

- **边界扫描（Boundary Scan）：** 用于生产线上的硬件自检。它可以检测芯片引脚是否焊接良好，线路是否存在短路或断路，而不需要用探针去物理接触引脚。

- **固件烧录（Programming）：** 将编译好的固件（Bootloader、内核镜像等）直接烧写到芯片内部的 Flash 或外部存储器中。

## 2. JTAG 的标准信号线

标准的 JTAG 接口通常由 4 根信号线组成（有时外加一根复位线）：

| **信号名称** | **全称**         | **功能描述**                                        |
|--------------|------------------|-----------------------------------------------------|
| **TMS**      | Test Mode Select | **模式选择**。控制测试访问端口（TAP）状态机的切换。 |
| **TCK**      | Test Clock       | **测试时钟**。为数据传输提供同步时钟。              |
| **TDI**      | Test Data In     | **数据输入**。测试指令或数据由此进入芯片。          |
| **TDO**      | Test Data Out    | **数据输出**。芯片内部状态或数据由此传回调试器。    |
| **TRST**     | Test Reset       | **可选复位**。用于异步复位 JTAG 测试逻辑。          |

## 3. JTAG 的工作原理：菊花链（Daisy Chain）

JTAG 的一个强大之处在于它支持**串联**。如果你的一块板子上有多个芯片（比如一个 SoC 加一个 FPGA），你可以将它们的 JTAG 接口首尾相连，形成一个“菊花链”。这样，你只需要一个调试器插口，就可以同时访问板子上的所有芯片。

## 4. JTAG 与 SWD 的区别

在你接触的嵌入式开发（如 RK3568 或 ARM 架构芯片）中，你可能还会听说 **SWD** (Serial Wire Debug)：

- **JTAG：** 标准成熟，支持芯片多，支持菊花链，但占用引脚较多（至少 4 根）。

- **SWD：** ARM 公司推出的协议，只需要 **2 根线**（SWDIO 和 SWCLK），抗干扰能力强，在空间紧张的微控制器（MCU）开发中非常流行。

## 5. 实际应用场景

作为一个嵌入式工程师，你通常会用到 **J-Link**、**ST-Link** 或 **DAP-Link** 这样的仿真器，通过 JTAG 接口连接到你的开发板。当你的系统在 Linux 内核启动阶段崩了（Kernel Panic），串口（UART）又没打印出有用信息时，JTAG 就是你查看 CPU 当前指令指针和堆栈信息的最后手段。

你目前在处理 RK3568 还是 Zynq-7100 的底层调试？这两者的 JTAG 配置在 Vivado 或 SDK 工具里略有不同。
