当 GPIO 被配置为**模拟模式（Analog Mode）时，它会经历一场“脱胎换骨”的变化：引脚内部的所有数字电路**（包括输入缓冲器、输出驱动器、上下拉电阻）都会被彻底断开。此时，引脚直接通过一条专用的模拟通道连接到芯片内部的 **ADC（模数转换器）** 或 **DAC（数模转换器）** 模块。

以下是 GPIO 在模拟模式下配合 ADC 和 DAC 工作时的详细机制：

### 1. 模拟模式的底层状态：彻底的“解耦”

在数字模式下，GPIO 内部有施密特触发器来整理波形，有 MOS 管来推拉电平。但在模拟模式下：

- **关闭施密特触发器：** 否则它会将连续变化的模拟电压强制翻转为 0 或 1，导致信号失真。

- **断开上下拉电阻：** 内部电阻会与外部信号源形成分压，导致测量电压不准。

- **零功耗干扰：** 禁用数字输入补偿，可以消除数字电路切换时产生的电磁噪声（Noise），这对高精度的模拟采集至关重要。

### 2. GPIO 配合 ADC（模数转换）：听诊器模式

ADC 的作用是将外部的**连续电压**（比如 \$1.25\text{V}\$）转换成 CPU 能理解的**数字值**（比如 0x666）。

- **采样保持（Sample and Hold）：** 当 ADC 开始工作，GPIO 引脚内部的一个微小电容（采样电容）会与外部电路接通并充电，直到电容电压等于外部电压。

- **逐次逼近 (SAR) 逻辑：** 芯片内部会像“天平称重”一样，不断尝试不同的二进制组合，直到内部生成的参考电压与引脚电压对齐。

- **电气要求：**

  - **输入阻抗：** 模拟模式下引脚阻抗极高。但如果外部信号源阻抗太大，采样电容充电太慢，会导致读数偏小。

  - **电压范围：** 严禁超过 \$V\_{REF}\$（参考电压，通常是 \$3.3\text{V}\$），否则采样会“打顶”失真，甚至烧毁通道。

### 3. GPIO 配合 DAC（数模转换）：调色板模式

DAC 的作用正好相反：它根据 CPU 给出的数字（比如 4095），在 GPIO 引脚上产生对应的**模拟电压**（比如 \$3.3\text{V}\$）。

- **电阻网络分压：** DAC 内部通常由一系列高精度的电阻梯形网络（R-2R Network）组成。通过控制开关，将 \$V\_{REF}\$ 进行精确分压后引向 GPIO。

- **输出缓冲（Output Buffer）：** 由于 DAC 内部电阻网络很弱，直接接负载会导致电压塌陷。因此，模拟模式下的 GPIO 往往会开启一个内部的\*\*运放（Op-Amp）\*\*作为缓冲，增强电流驱动能力。

- **应用场景：**

  - **音频输出：** 产生音频波形驱动耳机。

  - **波形发生器：** 产生正弦波、三角波。

  - **偏置电压：** 为精密传感器提供稳定的参考基准。

### 4. 关键参数对比

| **特性**          | **ADC (Input Analog)**                 | **DAC (Output Analog)**                |
|-------------------|----------------------------------------|----------------------------------------|
| **数据流向**      | 外部 \$\rightarrow\$ 内部 (电压变数字) | 内部 \$\rightarrow\$ 外部 (数字变电压) |
| **分辨率**        | 通常为 10-bit / 12-bit (4096级)        | 通常为 8-bit / 12-bit                  |
| **采样率/转换率** | 每秒采样次数 (SPS)                     | 每秒更新电压次数                       |
| **核心挑战**      | **噪声抗干扰**（需要电容滤波）         | **驱动能力**（电压是否会因负载下降）   |

### 5. 在 Linux 驱动中的特殊身份：IIO 子系统

在 Linux 内核中，如果一个 GPIO 被用于 ADC 或 DAC，它就不再受 GPIO 子系统管辖，而是归属于 **IIO (Industrial I/O)** 子系统。

- **设备树配置：** 你需要定义 io-channels，指定哪个引脚连接到哪个 ADC 通道。

- **用户空间访问：** 通过 /sys/bus/iio/devices/iio:device0/in_voltageX_raw 读取原始数值。

### 总结

模拟模式下的 GPIO 就是一条**纯净的模拟通道**。ADC 像是一个精密的**电压表**在后台记录，而 DAC 像是一个受控的**可变电源**在对外供电。
