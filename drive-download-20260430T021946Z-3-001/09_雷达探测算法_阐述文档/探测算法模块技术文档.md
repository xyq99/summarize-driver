# 探测算法模块技术文档

## 1. 概述
该目录主要实现了基于声学传感器的目标探测算法。算法流程涵盖了从原始数据输入、数字下变频(DDC)、波束成形(Beamforming)、匹配滤波(Matched Filtering)到最终的目标检测与参数提取（CFAR检测）。算法深度依赖于特定平台的 DSP 库（如 NEON 加速指令集）。

## 2. 核心算法流程
算法的总体执行流程集成在 `detection_moudle.c` 的 `detection_moudle_run` 函数中，具体步骤如下：

1.  **数据接收 (`detection_moudle_recive`)**
    *   将 48 通道、128 采样点的原始数据解析并存入循环缓冲区。
2.  **下变频处理 (`downconv.c`)**
    *   **复数移频**：将信号从 4500Hz 载频下移至基带。
    *   **三级滤波降采样**：
        *   第一级：94 抽头 CIC 滤波器。
        *   第二级：9 抽头 CIC 滤波器。
        *   第三级：25 抽头 FIR 滤波器。
3.  **波束成形 (`cbf_beamformer.c`)**
    *   将 48 个传感器分为两个 24 元子阵处理。
    *   根据声速动态计算导向矢量 (sv) 和波束权重 (w)。
    *   通过复数加权求和形成 61x2 个方向的波束信号。
4.  **匹配滤波 (`mf1`)**
    *   对波束成形后的信号进行时域匹配滤波，增强信噪比。
5.  **CFAR检测 (`cfar.c`)**
    *   **两维滤波**：分别在行和列方向进行预滤波处理。
    *   **峰值提取**：检测行列匹配的极大峰值点。
    *   **恒虚警检测**：计算目标周围背景噪声，根据设定的虚警率门限确认目标。
6.  **结果输出**
    *   通过回调函数输出目标的 **dB值**、**水平角度** 和 **距离**。

## 3. DSP接口调用情况
该算法大量调用了底层 DSP 加速接口，主要调用包括：

| 接口名称 | 功能描述 | 主要应用位置 |
| :--- | :--- | :--- |
| `DSP_init()` | 初始化 DSP 库环境 | 模块注册阶段 |
| `DSP_sp_vv_cr()` | 复向量与实向量点乘 | 下变频滤波器 |
| `DSP_sp_v_sum()` | 向量元素求和 | 滤波器输出计算 |
| `DSP_sp_vv_cc_conj()` | 复向量与另一复向量共轭点乘 | 波束成形权重应用 |
| `NEON_fft_finalize()` | 清理 FFT 相关资源 | 模块卸载阶段 |
| `log10f()` / `cos()` / `sin()` | 标准浮点数学运算 | 参量提取与权重计算 |

## 4. 关键数据结构 (`detection_commom.h`)
*   **`detection_param_t`**: 全局参数结构体，包含了滤波状态 (`filter`)、波束权重 (`beam`)、检测结果 (`cfar`) 以及原始数据缓冲区 (`flank_data_detection`)。
*   **`CICFilterState`**: 存储各级滤波器的状态缓冲区及历史索引，支持流式实时处理。

## 5. 主要文件清单
| 文件名 | 职责描述 |
| :--- | :--- |
| `detection_moudle.c` | 模块入口、算法状态机调度、数据分发。 |
| `downconv.c` | 数字下变频核心实现，包含大量硬编码的滤波器系数。 |
| `cbf_beamformer.c` | 常规波束成形实现，包含子阵时延矩阵。 |
| `cfar.c` | 恒虚警检测逻辑，协同 `cfar_detector.c` 工作。 |
| `myfindpeaks.c` | 行列方向的峰值搜索算法。 |
| `detection_commom.h` | 定义了所有的宏常量（如通道数48、采样点128）和核心结构体。 |
