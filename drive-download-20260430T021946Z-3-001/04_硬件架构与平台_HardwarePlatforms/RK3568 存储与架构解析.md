针对 RK3568 这颗在工业控制、物联网网关和嵌入式领域非常流行的 SoC，理解其存储体系和架构布局是开发底层驱动（如 U-Boot 或 Kernel）的基础。

RK3568 采用的是典型的**多级存储架构**，旨在平衡性能、成本和非易失性。

## 1. RK3568 存储体系全解析

在 RK3568 上，存储设备按用途可以分为：**片内高速存储**、**一次性物理配置**、**系统运行内存**以及**持久化外存**。

### 片内存储 (On-Chip Storage)

- **ROM (Boot ROM):**

  - **位置：** 集成在 SoC 内部。

  - **作用：** 固化了厂家最底层的启动代码（即 MaskROM 阶段）。它无法被修改。

  - **应用：** 上电第一步运行。它负责扫描外部设备（eMMC, SD 卡, USB），寻找有效的引导程序（如 idbloader.img）并加载到 SRAM。

- **SRAM (Internal SRAM):**

  - **特点：** 速度极快，但容量很小（通常几百 KB）。

  - **应用：** 在 DDR 初始化之前，它是 CPU 唯一可用的随机存储器。U-Boot 的第一阶段（TPL/SPL）就在这里运行，其核心任务就是**初始化 DDR**。

- **OTP (One-Time Programmable):**

  - **特点：** 只能写入一次，掉电不丢失。

  - **应用：** 存放芯片唯一 ID、MAC 地址、安全启动（Secure Boot）的公钥哈希以及电压修整（Trim）数据。

### 片外存储 (Off-Chip Storage)

- **DDR (DDR3/DDR4/LPDDR4/LPDDR4X):**

  - **作用：** 运行时内存。RK3568 拥有 32-bit 位宽的 DDR 接口。

  - **地位：** 所有的代码执行（Linux Kernel, Android）和多媒体缓冲都在这里。

- **eMMC / SPI Flash / SD Card:**

  - **作用：** 持久化存储。

  - **区别：** eMMC 速度最快且稳定，通常存放操作系统固件；SPI Flash 常用于存放极精简的引导代码。

## 2. RK3568 芯片系统架构

RK3568 是一颗 **Quad-core Cortex-A55** 架构的 SoC，其设计亮点在于其强大的 I/O 扩展能力和多媒体性能。

### 核心处理层 (Processing Unit)

- **CPU:** 四核 ARM Cortex-A55，采用高性能、低功耗的私有 L3 缓存架构。

- **NPU:** 内置 1.0 TOPs 算力的 AI 加速器，专门处理轻量级深度学习模型。

- **GPU:** ARM Mali-G52，支持 OpenGL ES 3.2 和 Vulkan 1.1。

### 多媒体引擎 (Multimedia)

- **VPU:** 支持 4K 60fps H.264/H.265 硬解码和 1080p 60fps H.264/H.265 硬编码。这使得它在视频监控和商显设备上表现优异。

### 扩展总线 (High-Speed I/O) —— RK3568 的杀手锏

RK3568 之所以被称为“接口之王”，是因为其灵活的引脚复用架构：

- **PCIe 3.0:** 支持 1x2Lane 或 2x1Lane 模式。

- **SATA 3.0:** 最多支持 3 路 SATA。

- **USB 3.0:** 支持双 USB 3.0。

- **双千兆以太网:** 内部集成两个 GMAC。

## 3. 存储与架构的协同：启动流程

理解了上述存储器，就能理清 RK3568 是如何“活”过来的：

1.  **MaskROM 阶段：** 芯片上电，CPU 从内部 **ROM** 读取指令，初始化最基础的堆栈。

2.  **引导加载阶段：** MaskROM 将 eMMC/SD 卡中的引导代码加载到内部 **SRAM**。

3.  **DDR 初始化：** SRAM 中的代码运行，配置 **DDR** 控制器的参数（频率、时序）。

4.  **搬运与跳转：** 一旦 DDR 就绪，代码会将完整的 **U-Boot** 从 **eMMC** 搬运到 **DDR**。

5.  **内核启动：** U-Boot 最后将 **Linux Kernel** 读入 DDR，解压并跳转执行，系统正式进入 OS 环境。

## 总结

- **SRAM** 是 DDR 就绪前的临时“阵地”。

- **OTP** 是芯片的“身份证”和“保险箱”。

- **DDR** 是系统的“主战场”。

- **eMMC** 是系统的“仓库”。

RK3568 通过高效的内部总线将 A55 CPU 与这些存储资源连接，并凭借丰富的 PCIe/SATA 扩展接口，成为了目前国产嵌入式高性能 SoC 的代表。
