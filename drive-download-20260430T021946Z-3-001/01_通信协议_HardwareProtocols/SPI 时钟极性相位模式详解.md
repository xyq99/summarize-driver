SPI 的四种模式是由 **CPOL（时钟极性）** 和 **CPHA（时钟相位）** 这两个参数交叉组合而成的。它们决定了时钟线（SCK）在空闲时是什么电平，以及数据是在时钟的哪个边缘（上升沿或下降沿）被采样。

我们可以通过一个核心逻辑来拆解这四种模式。

## 一、 参数的核心含义

1.  **CPOL (Clock Polarity) - 决定“空闲”状态**

    - **CPOL = 0**：空闲时为 **低电平** (Low)。

    - **CPOL = 1**：空闲时为 **高电平** (High)。

2.  **CPHA (Clock Phase) - 决定“采样”时刻**

    - **CPHA = 0**：在时钟周期的 **第一个跳变沿** 采样。

    - **CPHA = 1**：在时钟周期的 **第二个跳变沿** 采样。

## 二、 四种模式归纳表

| **模式 (Mode)** | **CPOL** | **CPHA** | **空闲电平** | **采样边缘（读取数据）** | **切换边缘（更新数据）** |
|-----------------|----------|----------|--------------|--------------------------|--------------------------|
| **Mode 0**      | 0        | 0        | 低 (Low)     | **上升沿** (第一个沿)    | 下降沿                   |
| **Mode 1**      | 0        | 1        | 低 (Low)     | **下降沿** (第二个沿)    | 上升沿                   |
| **Mode 2**      | 1        | 0        | 高 (High)    | **下降沿** (第一个沿)    | 上升沿                   |
| **Mode 3**      | 1        | 1        | 高 (High)    | **上升沿** (第二个沿)    | 下降沿                   |

## 三、 深度总结与理解技巧

为了不混淆这四种模式，你可以记住以下两个“黄金准则”：

## 1. 采样沿的判定逻辑

- 如果 **CPOL 和 CPHA 相同**（00 或 11），采样点都在 **上升沿**。

- 如果 **CPOL 和 CPHA 不同**（01 或 10），采样点都在 **下降沿**。

## 2. CPHA=0 的特殊性（如你上传的图）

- 当 **CPHA = 0** 时，数据必须在时钟开始跳变 **之前** 就准备好。因为第一个沿就要采样，系统没有时间在时钟开始后再去切换数据。

- 当 **CPHA = 1** 时，数据是在第一个沿发生后才切换，在第二个沿才被采样，这给了硬件更多的“准备时间”。

## 四、 硬件开发中的实际建议

在你的 **RK3568** 驱动开发或者外设调试中：

1.  **查阅 Datasheet：** 传感器或 Flash 芯片的手册会明确标注支持哪种 Mode（通常 Mode 0 和 Mode 3 最常用）。

2.  **相位错位的后果：** 如果 Master 和 Slave 的模式不匹配（比如一个设为 Mode 0，另一个设为 Mode 1），最典型的现象是 **读到的数据发生了位移**（Bit Shift），即所有的 bit 往左或往右挪了一位，导致数值完全错误。

3.  **驱动强度与相位：** 在高速 SPI 通信中，如果驱动强度（Drive Strength）不足导致边沿变缓，Mode 0 这种在第一个沿采样的模式会比 Mode 1 更容易出错，因为它的数据准备时间（Setup Time）更紧迫。
