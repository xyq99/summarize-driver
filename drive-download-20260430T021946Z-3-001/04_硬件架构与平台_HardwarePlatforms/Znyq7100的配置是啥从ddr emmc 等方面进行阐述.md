Zynq-7100（XC7Z100）是 Xilinx Zynq-7000 系列中的“天花板”级别型号，属于 **All Programmable SoC**。它的核心架构是将双核 ARM 处理器（PS 端）与大规模的 FPGA 逻辑（PL 端）集成在单颗芯片上。

针对你提到的 DDR、eMMC 等存储和外设配置，我们可以从 **PS（处理系统）** 和 **PL（可编程逻辑）** 两个维度来拆解：

## 1. 核心架构：PS + PL

- **PS 端 (Processing System):** 搭载双核 **ARM Cortex-A9**，主频通常在 800MHz - 1GHz 之间。这是运行 Linux 内核、管理 SSH 和网络的核心。

- **PL 端 (Programmable Logic):** 拥有高达 **444K 的逻辑单元 (Logic Cells)**。这是 Zynq-7100 最强悍的地方，规模相当于 Kintex-7 K410T，适合处理高带宽、高吞吐量的并行数据。

## 2. 存储配置（DDR、eMMC 等）

### **DDR 内存配置**

Zynq-7100 的 PS 端内置了硬核内存控制器，支持多种动态内存：

- **类型：** 支持 **DDR3、DDR3L、DDR2** 或 **LPDDR2**。

- **位宽：** 最大支持 **32-bit** 数据位宽。

- **容量：** 虽然架构理论支持更大，但在嵌入式设计中，PS 端常见的配置是 **1GB 或 2GB DDR3**。

- **PL 端 DDR：** 既然是 7100，通常会在 PL 端通过高性能接口（HP）扩展额外的 DDR（称为 PL-DDR），用于存储高速采集的原始数据。

### **非易失性存储（eMMC / QSPI / SD）**

PS 端的存储控制器支持多种启动和存储介质：

- **eMMC：** 支持 eMMC v4.41 标准。在工业级设计中，通常用 eMMC 替代 SD 卡作为根文件系统的存储介质，因为它比 SD 卡更稳定且读写寿命更长。

- **QSPI Flash：** 通常配置 **16MB - 128MB**，用于存放第一阶段启动代码（FSBL）、比特流（Bitstream）和轻量级内核镜像。

- **SD/SDIO：** 支持 2 个 SDIO 接口。在开发阶段，通常通过 SD 卡启动系统；在生产阶段，则常用于数据扩展。

## 3. 外设接口与高速连接

Zynq-7100 相比你之前提到的 TI138，在性能和接口上是跨代级的提升：

- **千兆以太网 (GigE)：** PS 端集成 2 个硬核千兆网控器（支持 RGMII 接口）。

- **USB：** 支持 2 个 USB 2.0 OTG 接口。

- **高速串行收发器 (GTX)：** 这是 7100 的王牌。它拥有 **16 个 GTX Transceivers**，每个通道速率高达 **12.5 Gbps**。这使得它支持：

  - **PCIe Gen2 x8**

  - **SFP+ (10GbE)**

  - **SATA**

  - **SRIO**

## 4. 与 TI138 的对比差异

如果你从 TI138 转向 Zynq-7100，你会发现工作内容有几个明显变化：

| 特性         | TI OMAP-L138           | Zynq-7100                                               |
|--------------|------------------------|---------------------------------------------------------|
| **DDR 处理** | 单一 DDR 控制器        | PS 和 PL 可以各有独立的 DDR 空间                        |
| **启动流程** | U-Boot -\> Kernel      | FSBL -\> Bitstream -\> U-Boot -\> Kernel                |
| **外设扩展** | 受限于芯片引脚固定功能 | 只要 PL 资源够，可以用 FPGA 逻辑生成无限个 UART/SPI/CAN |
| **文件系统** | 通常较小（NOR/SD）     | 通常较大（eMMC/SD），支持更复杂的 Ubuntu/Debian 根系统  |

## 5. 工作中的具体应用

在 Zynq-7100 上进行移植时，你的任务通常包括：

1.  **Vivado 硬件配置：** 在 Vivado 软件里配置 PS 端的 DDR 参数（时序、容量）和 eMMC 控制器引脚。

2.  **生成 FSBL：** 这是 Zynq 特有的，负责初始化 DDR 和加载 FPGA 比特流。

3.  **设备树（DTS）编写：** 相比 TI138，Zynq 的设备树需要包含大量由 PL 端映射到 PS 端的地址空间（AXI 总线设备）。

**既然你手头有 16GB 的 eMMC 空间（类似 SD 卡），在 7100 上你可以跑非常完整的 Linux 系统。你目前的任务是打算把 TI138 上的旧业务迁移到 Zynq-7100 上吗？**
