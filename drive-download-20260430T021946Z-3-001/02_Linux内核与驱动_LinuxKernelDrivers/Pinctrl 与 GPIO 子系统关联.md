在 Linux 内核驱动开发中，**pinctrl (Pin Control)** 和 **GPIO (General Purpose Input/Output)** 子系统是紧密相连但分工明确的“搭档”。简单来说，pinctrl 负责引脚的“多功能配置”，而 GPIO 负责引脚的“逻辑电平输入输出”。

## 1. 核心分工：谁负责什么？

为了理解它们的关联，可以把 SoC 的引脚想象成一个多功能的插座：

### **pinctrl 子系统 (管脚控制)**

它是“复用控制中心”。现代 SoC 的引脚通常是复用的（Muxing），一个引脚既可以做 GPIO，也可以做 I2C 的 SDA，或者 PWM 输出。

- **引脚复用 (Muxing)：** 决定引脚当前的功能（是 GPIO、UART 还是 SPI？）。

- **电气特性 (Configuration)：** 设置引脚的上下拉电阻、驱动能力（电流大小）、开漏（Open-Drain）等。

### **GPIO 子系统 (通用输入输出)**

它是“逻辑控制器”。只有当 pinctrl 将引脚配置为 **GPIO 模式** 后，GPIO 子系统才能介入。

- **电平控制：** 设置高低电平（set_value）。

- **状态读取：** 读取输入电平（get_value）。

- **中断管理：** 将引脚配置为中断触发源。

## 2. 它们是如何“配合”的？

在内核中，两者的关联主要体现在**依赖关系**和**交互接口**上。

### **引脚申请的链路**

当你在驱动中申请一个 GPIO 时（例如使用 gpiod_get），内核底层会发生以下链式反应：

1.  **GPIO 子系统** 接收到请求。

2.  它发现该 GPIO 对应的物理引脚属于某个 **pinctrl 句柄**。

3.  **pinctrl 子系统** 介入，自动将该引脚切换为 GPIO 功能（如果硬件支持这种自动切换），并根据 Device Tree 的配置设置好上下拉。

4.  配置完成后，**GPIO 子系统** 才开始接管该引脚的电平读写。

### **设备树 (Device Tree) 中的体现**

在 RK3568 等芯片的开发中，你会发现 GPIO 节点通常引用了 pinctrl 的配置。例如：

> DTS

&uart2 {  
pinctrl-names = "default";  
pinctrl-0 = \<&uart2m1_xfer\>; // 告诉 pinctrl 将引脚切到 UART 模式  
status = "okay";  
};  
  
// 或者配置为 GPIO  
my_device {  
pinctrl-names = "default";  
pinctrl-0 = \<&my_gpio_pins\>; // 配置为 GPIO 模式且带上拉  
gpios = \<&gpio0 RK_PB5 GPIO_ACTIVE_HIGH\>;  
};

## 3. 关键关联接口：gpio_chip 与 pinctrl_dev

在底层驱动实现中，它们通过以下方式绑定：

- **Range 映射：** 芯片驱动会在初始化时调用 gpiochip_add_pin_range()。这个函数将 GPIO 端口的编号（如 GPIO0_A1）与 pinctrl 内部管理的物理引脚编号（Pin ID）建立一一映射关系。

- **互斥关系：** 如果一个引脚已经被 pinctrl 配置为 I2C 模式，GPIO 子系统尝试操作它时通常会失败（除非硬件允许），从而避免硬件冲突。

## 总结

| **特性**     | **pinctrl 子系统**              | **GPIO 子系统**                     |
|--------------|---------------------------------|-------------------------------------|
| **关注点**   | 引脚“是什么”（功能选择）        | 引脚“发什么”（高低电平）            |
| **配置内容** | 复用、上下拉、驱动强度          | 输入、输出、中断触发                |
| **执行顺序** | 必须先配置（通常在 probe 阶段） | 配置完成后进行逻辑操作              |
| **层级关系** | 处于更底层，是 GPIO 的基础      | 建立在 pinctrl 提供的 GPIO 功能之上 |

**一句话总结：** pinctrl 是\*\*“修路”**的（把引脚功能拨到 GPIO 这一边），GPIO 是**“开车”\*\*的（在路上跑高低电平信号）。
